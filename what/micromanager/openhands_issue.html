Using (small) local models with OpenHands

We have servers with 6GB GPUs that I would like to use local models to generate boilerplate code.
The servers are powerful enough to run Aider with small local models, but OpenHands malfunctions.

We checked https://github.com/OpenHands/OpenHands/issues for similar issues but did not find one.

We also read the OpenHands documentation related to using local models.

I suspect others have had issues with running (small) models locally.

Models we've tried:

  • devstral:24b
  • deepseek-r1:14b
  • deepcoder:14b
  • mistral:7b
  • deepseek-coder:6.7b
  • wizardcoder:7b-python
  • qwen3:8b
  • qwen3-coder:30b
Model Size Status
devstral 24b ✅ Works
deepseek-r1 14b ❌ Gets stuck in loop
deepcoder 14b ⚠️ Creates file but gets stuck
mistral 7b ❌ Doesn't work
deepseek-coder 6.7b ❌ Doesn't work
wizardcoder 7b-python ❌ Doesn't work
qwen3 8b ⚠️ Somewhat worked
qwen3-coder 30b ❌ Doesn't work

The smaller models either didn't work or only somewhat worked. If I recall correctly it was qwen3:8b that somewhat worked. Being larger in size didn't necessarily mean the model worked, but of the models that did the job and didn't encounter issues, they were 14b+ ones. The primary issue I recall encountering was the models not understanding that a back and forth conversation was taking place.

They didn't understand:

> Please continue on whatever approach you think is suitable. 
  If you think you have solved the task, please finish the interaction.

There is also the models not knowing about or having the ability to interact with local files through the agent, such as:

> I'm sorry for the confusion, but as an AI model developed by OpenAI, 
  I don't have access to a runtime environment or any specific hosts. 
  My main function is to provide information and answer questions related 
  to programming and computer science based on my training data.
